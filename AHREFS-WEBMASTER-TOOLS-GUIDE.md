# Ahrefs Webmaster Tools (бесплатная версия) — Полное руководство

> Составлено на основе рекомендаций SEO-экспертов (SEOSLY, Search Engine Journal, Rankinity, Scholastica и др.)
> Фокус: бесплатный тариф AWT для нового академического журнального сайта (AIR)

---

## 1. Что входит в бесплатный тариф

| Инструмент | Лимиты |
|---|---|
| **Site Explorer** | 1 000 строк на отчёт, 100 000 строк/мес |
| **Site Audit** | 5 000 URL/проект/мес, данные хранятся 3 месяца |
| **Web Analytics** | До 1 млн событий/мес |
| **Social Media Manager** | 3 канала, 10 постов/мес |
| **SEO Toolbar** (расширение) | Базовые функции (редиректы, on-page аудит) |
| **AI Writing Tools** | 3 использования/день |
| **GBP Monitor** | Без ограничений |

**Верификация сайта** — обязательна. Три способа:
1. Импорт из Google Search Console (самый быстрый — 15 сек)
2. HTML мета-тег в `<head>`
3. Google Tag Manager

---

## 2. Первоначальная настройка

### Настройка Site Audit

- **Источники URL**: включить Website + Auto-detected sitemaps
- **Опции краулинга**: включить проверку images, CSS, JavaScript; follow non-canonical pages; follow nofollow links; remove URL parameters (предотвращает дубли)
- **Скорость краулинга**: оставить по умолчанию (~30 URL/мин)
- **НЕ включать** опцию "backlinks" в источниках — съедает кредиты

### Расписание аудитов

| Размер сайта | Рекомендация |
|---|---|
| До 500 страниц | Ежедневный аудит возможен |
| До 1 500 страниц | Еженедельный аудит |
| 1 500+ страниц | Ежемесячный полный краул + еженедельная проверка sitemap |

**Для AIR** (сейчас < 100 страниц): можно ставить ежедневный аудит без проблем.

---

## 3. Health Score — как интерпретировать

Health Score (0–100) — общий индикатор технического здоровья сайта.

- **Не гонитесь за числом** — важнее устранять Errors
- Score автоматически растёт при исправлении критических ошибок
- Средний хороший показатель для нового сайта: 80+

---

## 4. Приоритезация проблем (что чинить первым)

### Критические ошибки (красные) — чинить НЕМЕДЛЕННО
- 404/4XX битые страницы, на которые есть ссылки
- Страницы с таймаутом при краулинге
- Внутренние ссылки на несуществующие URL
- Orphan pages (страницы без входящих внутренних ссылок)

### Предупреждения (жёлтые) — важные улучшения
- Цепочки редиректов (301 → 301 → 301...)
- Отсутствующие или плохо написанные meta title / description
- Отсутствующий H1
- Медленные страницы
- Страницы, не включённые в sitemap

### Уведомления (синие) — полировка
- Страницы с только одной внутренней ссылкой
- Отсутствующие Open Graph теги
- Дубликаты без canonical
- Проблемы организации sitemap

---

## 5. Быстрые победы (Quick Wins)

1. **Восстановить/перенаправить битые URL** — мгновенно улучшает краулинг
2. **Обновить внутренние ссылки** — вместо ссылок через редиректы указать прямые URL
3. **Добавить внутренние ссылки на orphan pages** — улучшает обнаружение
4. **Оптимизировать title tags** (50–60 символов) и **meta descriptions** (140–160 символов)
5. **Найти ключевые слова на позициях 7–15** и доработать контент этих страниц (самый быстрый рост)
6. **Ключевые слова на позициях 11–20** — "low-hanging fruit" для перехода на первую страницу

---

## 6. Что отслеживать в Site Explorer

### Бэклинки
- Типы ссылок: dofollow, nofollow, UGC, sponsored
- Виды: in-content, image, redirect, canonical
- Domain Rating (DR) каждого линка
- Даты first seen / last seen
- **Фильтрация**: по одной ссылке на домен, по типу, хронологически
- **Действие**: сортировать по свежим ссылкам ежедневно для мониторинга

### Органические ключевые слова
- Позиция, объём, KD (сложность), CPC, трафик
- SERP features
- **Действие**: фильтровать по позициям 7–15 для поиска возможностей оптимизации

### Конкуренты
- Топ-10 конкурирующих доменов/страниц (бесплатно только 10)
- Общие и уникальные ключевые слова
- **Действие**: изучать темы конкурентов для вдохновения контента

### Лучший контент
- Top pages по ссылкам и росту ссылок
- Top keyword на странице
- **Действие**: определить 5–10 лучших страниц, реплицировать паттерны успеха

## 7. Ahrefs vs Google Search Console — зачем использовать оба

| Аспект | Google Search Console | Ahrefs Webmaster Tools |
|---|---|---|
| **Данные о кликах/показах** | Официальные от Google | Оценочные (estimates) |
| **Ключевые слова** | Реальные запросы с CTR | Расширенная база + KD + CPC |
| **Бэклинки** | Только подтверждённые Google | Собственный индекс, больше данных, фильтрация |
| **Core Web Vitals** | Да | Нет |
| **Технический аудит** | Базовый (индексация, ошибки) | 170+ проверок, категоризация по серьёзности |
| **Конкурентный анализ** | Нет | Да (топ-10 конкурентов бесплатно) |
| **Domain Rating** | Нет | Да |
| **Трафик от AI** | Нет | Да (Web Analytics) |

### Вывод
- **GSC** — для реальных данных о поисковой видимости, CTR, Core Web Vitals
- **Ahrefs** — для технического аудита, анализа бэклинков, конкурентов, и поиска возможностей роста
- **Используйте оба**: GSC для точных данных, Ahrefs для аналитики и стратегии

## 8. Типичные ошибки (чего НЕ делать)

1. **Ежедневные аудиты большого сайта** — быстро исчерпывают 5 000 кредитов/мес
2. **Чинить Notices вместо Errors** — всегда начинать с красных ошибок
3. **Дизавуить все спамные бэклинки** — обычно не нужно, Google сам их игнорирует
4. **Использовать AWT для анализа конкурентов** — бесплатный тариф показывает данные только по вашему верифицированному сайту
5. **Думать, что Web Analytics заменяет GSC** — это разные инструменты
6. **Доверять оценкам трафика Ahrefs как точным цифрам** — это estimates, реальные данные только в GSC/GA
7. **Игнорировать orphan pages** — страницы без внутренних ссылок = невидимы для поисковиков
8. **Не настроить расписание аудита** — без него данные устаревают

## 9. Рекомендуемый рабочий процесс

### Еженедельно (10–15 минут)
1. Открыть Site Audit → проверить Health Score (рост/падение)
2. Просмотреть новые Errors → создать задачи на исправление
3. Проверить новые бэклинки в Site Explorer (сортировка по "First seen")
4. Глянуть органические ключевые слова — есть ли новые ранжирования?

### Ежемесячно (30–45 минут)
1. Полный обзор Site Audit — сравнить с прошлым месяцем
2. Анализ конкурентов (топ-10 доменов) — какие темы они покрывают?
3. Обзор "Top pages" — какой контент привлекает ссылки?
4. Проверить ключевые слова на позициях 7–20 → спланировать доработку контента
5. Экспорт данных для отчёта / планирования

### Ежеквартально (1–2 часа)
1. Полный аудит + сравнение Health Score за 3 месяца
2. Стратегический обзор бэклинк-профиля
3. Обновление контент-стратегии на основе обнаруженных ключевых слов
4. Проверка и обновление sitemap

## 10. Специфические рекомендации для академического журнала (AIR)

### Технические приоритеты
- **HTML > PDF**: публиковать статьи в HTML формате (с отдельной страницей на каждую статью). PDF ранжируется ниже — Google предпочитает HTML с структурированными данными
- **Schema Markup**: добавить ScholarlyArticle schema на страницы статей (author, datePublished, headline, abstract, citation)
- **DOI ссылки**: все references в статьях должны содержать ссылки на DOI — это создаёт ценные исходящие ссылки
- **ORCID**: указывать ORCID авторов в метаданных
- **Meta tags для академики**: journal title, ISSN, publisher, volume, issue, publication date в HTML meta
- **Sitemap**: XML sitemap со всеми опубликованными статьями, обновлять при каждой публикации

### Контент и SEO
- **Заголовки статей**: должны содержать ключевые слова, понятно описывать тему (без хайпа)
- **Abstracts**: критически важны — некоторые академические поисковики индексируют только title + abstract
- **Keywords**: минимум 5 ключевых слов на статью, включать в метаданные
- **Rolling publication**: публиковать статьи по мере готовности (не ждать "выпуск") — поисковые боты чаще заходят на сайты с частым обновлением

### Построение ссылочного профиля
- **Цитирования = бэклинки**: в академии цитирования — самая мощная форма ссылок
- **DOAJ и каталоги**: подать заявку в Directory of Open Access Journals и другие индексы
- **Университетские домены**: ссылки с .edu доменов имеют высокую ценность
- **ResearchGate / Academia.edu**: авторы должны размещать ссылки на свои статьи
- **Google Scholar**: обеспечить правильную индексацию — требует корректных метаданных (Highwire Press tags или Dublin Core)

### Что отслеживать в Ahrefs для AIR
1. **Новые бэклинки** — особенно с .edu, .gov, .org доменов
2. **Ключевые слова** — по каким запросам находят статьи
3. **Orphan pages** — все статьи должны быть связаны внутренними ссылками
4. **Битые ссылки** — DOI ссылки могут устаревать
5. **Health Score** — держать выше 80
6. **Referring domains** — рост = хороший сигнал авторитетности

## 11. Чек-лист первоначальной настройки AWT для AIR

- [ ] Зарегистрировать аккаунт на ahrefs.com/webmaster-tools
- [ ] Верифицировать americanimpactreview.com (через GSC импорт — быстрее всего)
- [ ] Настроить Site Audit: включить Website + Sitemaps, проверку images/CSS/JS
- [ ] Запустить первый аудит
- [ ] Установить расписание: ежедневный краул (сайт маленький, кредитов хватит)
- [ ] Просмотреть результаты: Errors → Warnings → Notices
- [ ] Исправить все критические ошибки
- [ ] Перезапустить аудит для проверки
- [ ] Настроить Web Analytics (установить скрипт на сайт)
- [ ] Установить SEO Toolbar в браузер
- [ ] Подключить GSC параллельно (если ещё не сделано)
- [ ] Добавить Schema Markup ScholarlyArticle на страницы статей
- [ ] Проверить наличие XML sitemap и его корректность
- [ ] Отправить sitemap в GSC
